# Why?

When you have a lot of conditional rendering logic, it can be hard to follow the flow of your component.
A render map is a way to streamline this logic and make it easier to follow.

Take, for instance, this `Button` component:

```typescript
const Button = ({ variant, children }) => {
	if (variant === "primary") {
		return <PrimaryButton>{children}</PrimaryButton>;
	}

	if (variant === "secondary") {
		return <SecondaryButton>{children}</SecondaryButton>;
	}

	if (variant === "tertiary") {
		return <TertiaryButton>{children}</TertiaryButton>;
	}

	if (variant === "quaternary") {
		return <QuaternaryButton>{children}</QuaternaryButton>;
	}

	return null;
};
```

# Render map

Instead, create a render map:

```typescript
enum Variant {
	Primary = "primary",
	Secondary = "secondary",
	Tertiary = "tertiary",
	Quaternary = "quaternary",
}

const numVariants = 4;

const renderMap = {
	[Variant.Primary]: PrimaryButton,
	[Variant.Secondary]: SecondaryButton,
	[Variant.Tertiary]: TertiaryButton,
	[Variant.Quaternary]: QuaternaryButton,
};

const Button = ({ variant, children }) => {
	// important part
	const Component = renderMap[variant];

	return <Component>{children}</Component>;
};
```

Boom! Now you have a single place to look for all of your render logic.
